# 작업 계획: 요약 기능 개선

## 목표

메모 요약 결과를 화면을 벗어나도 유지하고, 요약 문구를 존댓말로 생성되도록 개선합니다.

## 구현 단계

1. **요약 캐시 유틸 추가**  

- `src/utils/summaryCache.ts`를 생성하여 `localStorage` 기반 get/set 헬퍼 제공 (메모 ID별 요약 텍스트 저장/조회/삭제).

2. **Gemini 프롬프트 개선**  

- `src/lib/gemini.ts`의 `generateSummary` 프롬프트에 "존댓말"로 작성하도록 명시하여 결과 톤을 보장.

3. **MemoDetailModal 상태 연동**  

- 모달 열릴 때 해당 메모 ID의 캐시 요약을 불러와 표시.
- 새 요약 생성 시 캐시에 저장, 재진입 시 자동 재사용.
- 메모가 변경되면 이전 상태 초기화, 삭제 시 캐시 정리 등 안정성 점검.

## 확인 사항

- 캐시 만료 전략이 필요한가요, 아니면 무기한 저장해도 될까요?
