# 자동 태그 생성 계획

## 1. 태그 생성 로직 설계

- 기존 Gemini 요약 API 요청에 태그 생성을 포함하도록 프롬프트를 확장하고 응답 형식을 `{ summary, tags }`로 정의한다.
- API 라우트(`/api/memos/[id]/summary`)에서 새 태그 배열을 파싱하고 Supabase `tags` 필드에 병합/대체 여부를 결정한다(요구사항에 따라 기존 태그를 덮어쓰는 기본 전략으로 진행).

## 2. 훅 및 UI 연동

- `useMemos`에서 메모 생성/수정 후 요약 API를 호출해 자동 태그를 가져오고, Supabase에 업데이트한다.
- MemoForm 제출 시 태그 입력은 여전히 허용하되, 자동 생성 태그와 병합 규칙(예: 중복 제거 후 합치기)을 구현한다.
- MemoDetailModal과 MemoList가 새로운 태그 값을 즉시 반영하도록 상태를 갱신한다.

## 3. 프론트엔드 피드백 및 브라우저 검증

- 태그 자동 생성 과정에서 로딩 상태 또는 토스트 메시지를 표시해 사용자에게 진행 상황을 알린다.
- 필요한 경우 브라우저를 열어 메모 생성/수정 → 자동 태그 반영 시나리오를 실제로 확인하고 스크린샷 또는 설명을 제공한다.

## 4. 문서 및 환경 업데이트

- README에 “메모 저장 후 자동 태그 생성” 흐름과 주의사항을 기록한다.
- 추가 환경 변수나 설정이 필요하다면 `.env.example`에 반영한다.
